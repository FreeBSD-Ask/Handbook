# 30.10. 用于 Microsoft® Windows® 客户端的文件和打印服务（Samba）
Samba是一个使用SMB/CIFS协议，提供共享文件以及打印服务的开源软件包。SMB/CIFS协议已经内嵌于Microsoft®️ Windows®️操作系统。通过安装Samba客户端库也可以将其安装到非Microsoft®️ Windows®️操作系统上。该协议允许客户端访问共享数据以及打印机。这些共享可以被映射为一个本地磁盘驱动器，且允许您像使用本地打印机一样使用共享打印机。

在FreeBSD，可以使用port [net/samba413](https://cgit.freebsd.org/ports/tree/net/samba413/pkg-descr)或package安装Samba客户端库。使用该客户端，FreeBSD系统可以在一个Microsoft®️ Windows®️网络上访问SMB/CIFS共享网络。

同样，FreeBSD也可以被配置为一个Samba服务器。FreeBSD系统上的管理员可以创建一个SMB/CIFS共享，使其可以被Microsoft®️ Windows®️客户端或Samba客户端访问。

## 30.10.1. 配置服务器
Samba配置文件位于/usr/local/etc/smb4.conf 。在启动Samba服务之前，该文件必须被创建。

这里列出了一个简单的smb4.conf文件可以让工作组内的Windows®️客户端访问共享目录和打印机。对于LDAP及活动目录等更复杂的设置，可以使用[samba-tool(8)](https://www.freebsd.org/cgi/man.cgi?query=samba-tool&sektion=8&format=html)更容易地创建初始**smb4.conf**文件。
```
[global]
workgroup = WORKGROUP
server string = Samba Server Version %v
netbios name = ExampleMachine
wins support = Yes
security = user
passdb backend = tdbsam

# 示范: /usr/src 只向'developer'用户分享
[src]
path = /usr/src
valid users = developer
writable  = yes
browsable = yes
read only = no
guest ok = no
public = no
create mask = 0666
directory mask = 0755
```
### 30.10.1.1 全局设置
**/usr/local/etc/smb4.conf**中关于网络的设置：

***workgroup***  
&#x2003;要服务的工作组的名称 

***netbios name***  
&#x2003;Samba服务器已知的NetBIOS名称。 默认情况下，它与主机DNS名称的第一个组成字符相同。

***server string***  
&#x2003;该字符串将会在`netview`的输出及其他显示服务器信息的工具中显示。 

***wins support***  
&#x2003;是否使Samba成为WINS服务器。不要在服务器数量多于一个的网络上开启WINS支持。

### 30.10.1.2 安全设置
安全模型和后端密码格式是 **/usr/local/etc/smb4.conf** 中最重要的设置。指令与选项:

***security***  
&#x2003;`security = share`和`security = user`是最常见的设置。如果客户端和Freebsd机器上使用同样的用户名，则应当启用用户级别的安全性。这也是默认的安全策略。客户端必须登录才可以访问共享资源。  
&#x2003;在分享级别的安全性中，客户端不需要输入有效的登录凭证即可访问共享资源。这也是版本较老的Samba使用的默认的安全策略。  

***passdb backend***  
&#x2003;Samba有好几种不同的后端身份验证模型。客户端可以使用LDAP，NIS +，SQL数据库或编辑过的的密码文件进行身份验证。TDBSAM是简单网络的理想选择（上方配置文件使用）。 对于较大及更复杂的网络，建议使用LDAPSAM。SMBPasswd是老版本的默认选项，现在已经过时了。  

### 30.10.1.3 Samba中的用户
FreeBSD用户必须要映射到Windows®客户端上的`SambasamAccount`数据库才能访问共享。使用[pdbedit(8)](https://www.freebsd.org/cgi/man.cgi?query=pdbedit&sektion=8&format=html)映射现有FreeBSD用户账号:
```
# pdbedit -a 用户名
```
该章节仅提及常见的用户设置。有关可用配置选项的其他信息，请参阅[Samba官方Wiki](https://wiki.samba.org/index.php/Main_Page)。

## 30.10.2 启动Samba服务
若想要在开机时自动启动Samba，请在 **/etc/rc.conf** 中添加该行：
```
samba_server_enable="YES"
```

启动Samba:
```
# service samba_server start
Performing sanity check on Samba configuration: OK
Starting nmbd.
Starting smbd.
```
Samba由三个单独的守护进程组成。nmbd和smbd由Samba_Enable启动。如果需要提供Winbind名称解析，请设置：
```
winbindd_enable="YES"
```
Samba可以随时通过该命令停止：
```
# service samba_server stop
```
Samba是一个提供集成Microsoft®️ Windows®️网络功能的复杂套件。有关此处描述基本配置之外功能的更多信息，请参阅https://www.samba.org 。

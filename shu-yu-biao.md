`ACPI Machine Language`

伪代码，由 ACPI 兼容操作系统内的虚拟机解释执行，在底层硬件与操作系统文档化接口之间提供一层抽象。

`ACPI Source Language`

编写 AML 所用的编程语言。

`Access Control List`

附加到某个对象（通常是文件或网络设备）上的一组权限。

`Advanced Configuration and Power Interface`

一种规范，将硬件向操作系统呈现的接口抽象化，使操作系统无需了解底层硬件即可充分利用其功能。ACPI 演进并取代了此前由 APM、PNPBIOS 及其他技术提供的功能，提供了功耗控制、系统挂起、设备启用与禁用等能力。

`Application Programming Interface`

一组过程、协议和工具，规定了程序各部分之间的标准交互方式：它们如何协同工作、何时协同以及为何协同，以及它们共享或操作哪些数据。

`Advanced Power Management`

一套 API，让操作系统与 BIOS 协同工作以实现电源管理。在大多数应用场景中，APM 已由更通用、功能更强大的 ACPI 规范所取代。

`Advanced Programmable Interrupt Controller`

高级可编程中断控制器。

`Advanced Technology Attachment`

高级技术附件。

`Asynchronous Transfer Mode`

异步传输模式。

`Authenticated Post Office Protocol`

认证邮局协议。

`Automatic Mount Daemon`

一种守护进程，在访问文件系统内的文件或目录时自动挂载该文件系统。

`Basic Input/Output System`

`Berkeley Internet Name Domain`

`Berkeley Software Distribution`

`Bikeshed Building`

`Giant`

一种互斥机制（睡眠 mutex）的名称，用于保护大量内核资源。在机器仅有几十个进程、一块网卡、且只有一个处理器的年代，简单的锁定机制尚可应付。但到了今天，这已成为难以接受的性能瓶颈。FreeBSD 开发者正积极将其替换为保护单个资源的锁，这将让单处理器和多处理器机器都能实现更高程度的并行。

随着 FreeBSD 的发展，用户可见的变更应尽可能不令人惊讶。例如，在 **/etc/defaults/rc.conf** 中随意重新排列系统启动变量就违反了 POLA。开发者在考虑用户可见的系统变更时会遵循 POLA。

`PRIME`

一种通过共享直接内存访问（DMA）缓冲区使多个物理图形协处理器共存的方法。

`Problem Report`

对 FreeBSD 源代码或文档中发现的某种问题的描述。请参阅如何撰写 FreeBSD 问题报告。

`Process ID`

系统中每个进程独有的编号，用于标识该进程并对其执行操作。

`Project Evil`

NDISulator 的工作代号，由 Bill Paul 编写并命名——他如此命名，是指从哲学角度来看，竟然需要这样的东西本身就是一件糟糕的事。NDISulator 是一个特殊的兼容模块，让 Microsoft Windows NDIS miniport 网络驱动程序能在 FreeBSD/i386 上使用。这通常是使用闭源驱动程序网卡的唯一途径。参见 **src/sys/compat/ndis/subr_ndis.c**。
